# 案例研究：云游 Yunyou

> 一个机票决策平台的 MVP 复盘 —— 从灵感到上线的 21 天。

## 背景

**Iris** 作为 AI 产品方向的学生，想验证一个假设：*信息过载时代，用户需要的不是更多选项，而是更少的好选项*。

## 方案

1. 用户描述出行偏好（自然语言）
2. 豆包 API 结构化意图
3. 调用机票 API 返回候选
4. AI 给出 **三选一 + 理由**

## 数据

| 指标 | MVP 第 1 周 | MVP 第 3 周 |
|------|-------------|-------------|
| 日活 | 12 | 87 |
| 转化率 | 4% | 18% |
| NPS | 31 | 52 |

## 技术栈

| 层 | 工具 |
|----|------|
| 前端 | React + Vite |
| 部署 | Vercel |
| AI | 豆包 ARK |

def rank_flights(options, prefs):

## 结论

MVP 的价值不在功能完整，而在 *把假设送到用户面前*。
